Class IoCManager

The IoCManager handles Dependency Injection in the project.

Inheritance
System.Object
IoCManager
Namespace: Robust.Shared.IoC
Assembly: Robust.Shared.dll
Syntax
public static class IoCManager : object
Remarks

Dependency Injection is a concept where instead of saying "I need the EntityManager", you say "I need something that implements IEntityManager". This decouples the various systems into swappable components that have standardized interfaces.

This is useful for a couple of things. Firstly, it allows the shared code to request the client or server code implicitly, without hacks. Secondly, it's very useful for unit tests as we can replace components to test things.

To use the IoCManager, it first needs some types registered through Register<TInterface, TImplementation>(Boolean). These implementations can then be fetched with Resolve<T>(), or through field injection with DependencyAttribute.

IoCManager is actually a static wrapper class around a thread local IDependencyCollection. As such, IoCManager will not work in other threads, unless they have first been initialized with InitThread() or InitThread(IDependencyCollection, Boolean). You should not initialize IoC in thread pools like that of , since said thread pool might be used by different running instances (for example, server and client running in the same process, they have a different IoC instance).
